<h1>proposal-url-mirror-sheme-txt</h1>
<p>PROPOSAL URL mirror sheme AKA's "metalink URL"<br></p><p>v2.01 Metalink Discussion Google Group (October 7 2007)</p><p>maintained by sebastien@willlemijns.com</p><br><p>Permit to include two or more sources in a URL for best QoS<br></p><p>&nbsp;</p><p>sheme purposed:</p><p>-------------------------&nbsp;</p><p>&nbsp;</p><p>adding a lot of mirror URL in the same line for:</p><p>- using mirrors if main site fallback (network or server error)</p><p>- using mirrors servers in same time for quick surf<br></p><p><br>&nbsp;</p><p>examples:&nbsp;</p><p>-------------<br>&nbsp;</p><p>metalink:"<a rel="nofollow" href="http://ww.cnn.com/">http://www.cnn.com/</a>""<a rel="nofollow" href="http://ww.backup.com/cnn/">http://www.backup.com/cnn/</a>"&nbsp;</p><p><a rel="nofollow" href="http://ww.backup.com/cnn/">http://www.backup.com/cnn/</a> is equal to well known CNN website and can be use if main site is down or</p><p>unreached... this method can be used too to increase surf because you can use the two sites in same time&nbsp;</p><p>&nbsp;</p><p>- best choice is to privilegiate main site for dynamic contents ;)&nbsp;</p><p>- the metalink URL concept also suggest you to mirror all your static contents...<br> </p><p>&nbsp;</p>
<p>metalink:"<a rel="nofollow" href="http://ww.cnn.com/">http://www.cnn.com/</a>news.htm""<a rel="nofollow" href="http://ww.backup.com/cnn/">http://www.backup.com/cnn/</a>"</p><p><a rel="nofollow" href="http://ww.cnn.com/">http://www.cnn.com/</a>news.htm can be found at <a rel="nofollow" href="http://ww.backup.com/cnn/">http://www.backup.com/cnn/</a>news.htm</p><p>notice the filename has been forgotten in the mirror URL to avoid dupe informations</p><p>&nbsp;</p><p>metalink:"<a rel="nofollow" href="http://ww.cnn.com/">http://www.cnn.com/world/europe/france/</a>news.htm""<a rel="nofollow" href="http://ww.backup.com/cnn/">http://www.backup.com/french-news-cnn/</a>"</p>
<p><a rel="nofollow" href="http://ww.cnn.com/">http://www.cnn.com/world/europe/france/</a>news.htm can be found at <a rel="nofollow" href="http://ww.backup.com/cnn/">http://www.backup.com/french-news-cnn</a><a rel="nofollow" href="http://ww.cnn.com/">/</a>news.htm</p>

<p>this possibility of mixing protocols is great for downloading files as metalinks for example ;)</p><p>as first example you can use the two protocols in same time for increasing speed...<br></p><p>&nbsp;</p><p>limitations:</p><p>---------------&nbsp;</p><p>&nbsp;</p>- Size of URL is unlimited (actually it does but i prefer to add this info),
<p>&nbsp;</p><p><b>- URL can not contains "%22%22" , "<span>#x22;</span><span>#x22;</span>" (unicode hexadecimal), "<span>#34;</span><span>#34;" (unicode decimal)</span></b></p><p><b>for format string reason !!!</b> of course, other kind of characters normalizations which equals to 2 quotes&nbsp;</p><p>can not been used ;)<br></p><p>&nbsp;</p><p>- All mirrors URLs MUST contains a protocol on it</p><p><b> so this URL....&nbsp;&nbsp;&nbsp; metalink:"<a rel="nofollow" href="http://ww.cnn.com/">www.cnn.com/</a>""<a rel="nofollow" href="http://ww.backup.com/cnn/">www.backup.com/cnn/</a>"&nbsp;&nbsp; ..... is invalid !</b></p><p><b>&nbsp;</b><br></p><p>Discussions&nbsp;</p><p>&nbsp;</p><p>- How to avoid "non-dupe download effect" when this "metalink URL" is used for increasing download</p><p>with the helps of main site and all the mirrors in same time ?
