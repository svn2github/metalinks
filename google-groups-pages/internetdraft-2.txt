<h1>client</h1>
The latest version of the Internet Draft of the Metalink Download Description is at <a href="http://tools.ietf.org/html/draft-bryan-metalink">http://tools.ietf.org/html/draft-bryan-metalink</a><br>(Incremental updates are at <a href="http://metalinks.svn.sourceforge.net/viewvc/metalinks/internetdraft/">http://metalinks.svn.sourceforge.net/viewvc/metalinks/internetdraft/</a> )<br><br>Known issues:<br><ul><li>Section 5 - <a href="http://tools.ietf.org/html/draft-hammer-discovery">Autodiscovery</a> of Metalinks by Metalink Processors via <a href="metalink-discussiontransparent-metalinks">Transparent Content negotiation in HTTP</a> [RFC2295] vs <a href="http://tools.ietf.org/html/draft-nottingham-http-link-header">Web Linking (HTTP Link header)</a> [draft-nottingham-http-link-header]. <br></li></ul><br>&nbsp;&nbsp;&nbsp;&nbsp; Autodiscovery allows Metalink Processors to request a certain URI, without knowing there is a Metalink describing that URI, and then discover that there is a Metalink available at another URI.&nbsp; That is, a download client can<br>&nbsp;&nbsp;&nbsp;&nbsp; request a file over HTTP with a certain URI to start a traditional download, and discover there is a Metalink describing the file, then use the information in the Metalink to complete the download.<br><br>&nbsp;&nbsp;&nbsp;&nbsp; <a href="metalink-discussiontransparent-metalinks">Current usage documented</a>. Refer to this <a href="http://groups.google.com/group/metalink-discussion/browse_thread/thread/b59e5d99cf879529/e14baeaf51b69e3d?lnk=gst&amp;q=%22link+header%22#e14baeaf51b69e3d">discussion</a>. Out of 25+ download clients, only 4 (aria2, DTA, Metalink Checker, Retriever) support (experimental) Transparent Metalinks via TCN.<br><br>&nbsp;&nbsp;&nbsp;&nbsp; Web Linking (HTTP Link header) proposal:<br><div style="text-align: left;">&nbsp;&nbsp;&nbsp;&nbsp; Link: &lt;<a href="http://example.com/resource.metalink">http://example.com/resource.metalink</a>&gt;; rel="<span>describedby</span>"; type="application/metalink+xml";<br></div><br>&nbsp;&nbsp;&nbsp;&nbsp; Also see Eran Hammer-Lahav's analysis of <a href="http://tools.ietf.org/html/draft-hammer-discovery-03#appendix-B.2.3">HTTP Content Negotiation</a> and <a href="http://tools.ietf.org/html/draft-hammer-discovery-03#appendix-B.2.1">HTTP Response Header</a> of <a href="http://tools.ietf.org/html/draft-hammer-discovery">Link-based Resource Descriptor Discovery<span></span><span></span></a>.<br>&nbsp;&nbsp;&nbsp;&nbsp; In short, <br>&nbsp;&nbsp;&nbsp;&nbsp; <b>Transparent Content Negotiation</b>: <b>(+)</b> Less roundtrips/Less server load, <b>(-)</b> Firefox (and possibly others?) not willing to add to Accept: header, <b>(-)</b> not Web Architecture compliant. <br>&nbsp;&nbsp;&nbsp;&nbsp; <b>Link Header</b>: <b>(-)</b> More roundtrips/More server load, <b>(+)</b> Web Architecture compliant, <b>(- for now)</b> still a draft, not a standard.<br><br>Changes from current Metalink specification:<br><br><ul><li>Different namespace:&nbsp; xmlns="urn:ietf:params:xml:ns:metalink" (Previous: <span><span>xmlns</span>=<span>"http://www.metalinker.org")</span><span></span></span></li><li>Different date format: RFC 3339. Example: 2008-12-13T18:30:02Z</li><li>Section 5 - Autodiscovery has not previously been specified.<br></li><li>&lt;metalink&gt; attributes (published, origin, generator, updated) are now elements.</li><li>&lt;url&gt; no longer has type attribute listing ftp, http, bittorrent.<br></li><li>New &lt;metadata&gt; instead of URLs to torrents or metalinks.</li><li>Deprecated elements (&lt;changelog&gt;, others?)<br></li><li>IANA registry <a href="http://www.iana.org/assignments/hash-function-text-names/hash-function-text-names.xhtml">Hash Function Textual Names</a> defines values for hash types. The SHA hashes now have a "-" in them: sha-1, sha-224 sha-256, sha-384, sha-512 instead of "sha1" etc.<br></li><li>IANA registry <a href="http://www.iana.org/assignments/operating-system-names">Operating System Names</a> defines values for OS types. This registry would need to be updated.<br></li><li>Relax NG compact schema included.</li></ul><wbr>
