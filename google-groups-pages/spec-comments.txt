<h2>Change Date &amp; Time Format</h2><p>CURRENT: RFC 822, which I've been told has been obsolete since 2001 :)<br>Example: Mon, 15 May 2006 00:00:01 GMT</p><p>&nbsp;</p><p>PLANNED (switchover between 20 Feb and 05 Mar): <a href="http://en.wikipedia.org/wiki/ISO_8601">ISO 8601</a> / RFC 3339, which looks like [YYYY]-[MM]-[DD].<br>Example: 2008-01-09T16:33Z</p><p>Other uses: iCal (RFC 2445)</p><p>&nbsp;</p>&nbsp;<p>Note: This applies to the "releasedate" tag and "refreshdate" attribute.&nbsp;</p><h2>More detailed OS tag</h2><p>There should be a way to specify individual versions of Windows (98, NT, XP, Vista, etc). There should also be ways to specify the processor type (i386, arm, etc.) and number of bits (32 vs. 64)</p><p>&nbsp;</p><p>An already existing way to specify operating system versions in XML:<br><br>schema: <a href="http://nvd.nist.gov/schema/nvdcve.xsd">http://nvd.nist.gov/schema/nvdcve.xsd</a><br><br>&lt;prod name="Windows Vista" vendor="Microsoft"&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="32 bit" num="Ultimate"/&gt;<br>&lt;/prod&gt;<br><br>&lt;prod name="windows" vendor="Microsoft"&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp1" num="server_2003"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp2" num="server_2003"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="unknown" num="server_2003"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="unknown" num="server_2003"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp2" num="server_2003"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp1" num="server_2003"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp2" num="server_2003"/&gt;<br>&lt;/prod&gt;<br>&lt;prod name="windows-nt" vendor="Microsoft"&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp2" num="XP"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp1" num="XP"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="unknown" num="XP"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers edition="sp2" num="XP"/&gt;<br>&lt;/prod&gt;<br>&lt;prod name="Kernel" vendor="Linux"&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers num="2.4.0"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers num="2.4.0 test1"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers num="2.4.0 test10"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers num="2.4.0 test11"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers num="2.4.0 test12"/&gt;<br>&lt;/prod&gt;<br>&lt;prod name="Debian Linux" vendor="Debian"&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers num="4.0"/&gt;<br>&nbsp;&nbsp;&nbsp; &lt;vers num="4.0 alpha"/&gt;<br>&lt;/prod&gt;<br>&nbsp;</p><h2>Package Contents</h2><p>There should be a way of describing the contents of a download package, including file checksums. This can be useful for .exe installers, .zip, .tar.gz, etc. Suggest similar format to the &lt;file&gt; tag.</p><p><br></p><p>For reference, the current format used by Appupdater, we also probably want to specify a container tag similar to &lt;files&gt;:</p><pre>&lt;filename name="content.dll"&gt;<br>    &lt;sha1sum&gt;c052927a3f1082b90c35aaddfafb723fbfcc7c95&lt;/sha1sum&gt;<br>    &lt;md5sum&gt;eb94c3e1d5cdf93cad8e1aaff1aaa988&lt;/md5sum&gt;</pre><pre>&lt;/filename&gt;</pre><pre>or maybe</pre><p><font size="2">&lt;contents&gt;</font></p><p><font size="2">&nbsp; &lt;file name="content.dll"&gt;</font></p><p><font size="2">&nbsp;&nbsp;&nbsp; &lt;size&gt;4405843968&lt;/size&gt;<br>&nbsp;&nbsp;&nbsp; &lt;verification&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;hash type="md5"&gt;eb94c3e1d5cdf93cad8e1aaff1aaa988&lt;/hash&gt;<br>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; &lt;hash type="sha1"&gt;c052927a3f1082b90c35aaddfafb723fbfcc7c95&lt;/hash&gt;</font></p><p><font size="2">&nbsp;&nbsp;&nbsp; &lt;/verification&gt;</font></p><p><font size="2">&nbsp; &lt;/file&gt;</font></p><p><font size="2"><span style="font-family: courier new,monospace;">&lt;/contents&gt;</span></font><br></p><pre>&nbsp;</pre><h2>Namespace<br></h2><p>Use namespaces to allow for other custom tags to be included.</p><h2>Required Tags</h2><p>Change "identity" and "version" tags to required instead of recommended.
